Endpoints usados

  • `GET /api/routes` para rutas abiertas e historial.
  • `GET /api/orders` para candidatos de pedido, filtrando los que no pueden entrar a ruta.
  • `GET /api/routes/available-tandas` para tandas disponibles.
  • `POST /api/routes/preview` para calcular orden, distancia, tiempo y depot.
  • `POST /api/routes` para guardar la ruta. El backend optimiza de nuevo.
